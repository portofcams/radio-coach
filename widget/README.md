# Wilco — iOS home-screen widget (WidgetKit)

A small/medium home-screen widget showing a rotating **phonetic letter + radio
tip**, fully self-contained (no network, no App Group). Tapping deep-links to
`/practice`. Added to the iOS Xcode project by a **script** and verified to compile.

## Files
- `Sources/` (committed, durable):
  - `WilcoWidgetBundle.swift` — `@main` widget bundle
  - `WilcoWidget.swift` — `StaticConfiguration` + `TimelineProvider` + the view (hourly timeline, `.widgetURL`)
  - `Tips.swift` — the phonetic/tip content rotated by date
- `ios-template/` — `Info.plist` (NSExtension = widgetkit-extension) + a minimal
  `Assets.xcassets` (AccentColor + WidgetBackground).
- `../scripts/add-widget-target.rb` — stages the above into `ios/App/WilcoWidget`
  and adds the `WilcoWidget` app-extension target (sources, build settings,
  Info.plist, embed-in-PlugIns phase, shared scheme).

## Wiring it up
`ios/` is gitignored and regenerated by Capacitor, so re-run the script rather
than committing the project:
```sh
gem install xcodeproj            # once (shared with the watch script)
npx cap sync ios                 # ensure ios/ exists
ruby scripts/add-widget-target.rb
```
Verify it compiles (no signing needed):
```sh
xcodebuild -project ios/App/App.xcodeproj -target WilcoWidget \
  -sdk iphonesimulator -configuration Debug CODE_SIGNING_ALLOWED=NO build
# ** BUILD SUCCEEDED **   (verified 2026-06-20, Xcode 26.5)
```

## Before archiving
- Bundle id `com.binnacleai.radiocoach.widget`; `DEVELOPMENT_TEAM` is `CCSWC89S2V`
  with automatic signing — open the project once so Xcode makes the profile.
- The widget needs no special capability. It embeds in the App target's PlugIns,
  so archiving the **App** scheme ships it to TestFlight automatically.
- Deep links to `https://wilco.binnacleai.com/practice`; add Associated Domains to
  the app later if you want it to open the app instead of Safari.

## Notes
- Pure SwiftUI + WidgetKit; `containerBackground` is guarded for iOS 17+.
- Both the watch (`add-watch-target.rb`) and widget scripts can run on the same
  `ios/` project; they're independent.
