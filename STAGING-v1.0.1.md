# v1.0.1 — STAGED, ship AFTER Apple approves v1.0

Branch `feat/v1.0.1-clearspar-watch`, staged 2026-06-28 while **v1.0 (build 7) was
`WAITING_FOR_REVIEW`** (submitted 2026-06-24). **Do NOT submit this until v1.0 clears
review** — uploading a new build now pulls v1.0 out of Apple's queue and resets you.

## What this branch changes
1. **Watch app finally embeds.** `scripts/embed-watch-app.rb` (re)adds the "Embed Watch
   Content" build phase to the **`Wilco`** target — the main iOS app target is named
   `Wilco`, **NOT `App`**, which is why `add-watch-target.rb`'s embed step silently
   orphaned and the Watch app built-but-never-bundled in v1.0. Now WilcoWatch.app ships
   inside the iOS binary. Verified: `dstSubfolderSpec=16`, `dstPath=$(CONTENTS_FOLDER_PATH)/Watch`.
2. **Capacitor → clearspar.** `capacitor.config.ts` `server.url` is now
   `https://clearsparradio.binnacleai.com` (was wilco). The native webview loads the new domain.
3. **Version → 1.0.1, build 10** (across all targets; build 10 > 7 already uploaded).

## Apply + ship (when v1.0 = READY_FOR_SALE)
1. Confirm v1.0 is approved (ASC API: appStoreState).
2. `git checkout main && git merge feat/v1.0.1-clearspar-watch`
3. **Re-run `ruby scripts/embed-watch-app.rb`** (idempotent) — `ios/` can be regenerated by
   `cap`, so the script, not the pbxproj, is the source of truth for the embed phase + version.
4. `npx cap sync ios`, then archive + upload via the established altool path.
5. ASC: new 1.0.1 version → attach build 10 → submit.

## 301 wilco → clearspar — ONLY after 1.0.1 is LIVE
Until 1.0.1 is the live app, wilco MUST keep serving (v1.0 loads it; a 301 would break the
in-review/approved app's webview). Once 1.0.1 is live on clearspar:
- NPM API (admin pw in `/root/.npm-admin-pw`) → wilco proxy host is **id 42**.
- Set its `advanced_config` to `return 301 https://clearsparradio.binnacleai.com$request_uri;`
  (or convert it to a Redirection Host). Then wilco permanently 301s to clearspar.

## Why main is untouched
`main` stays at v1.0 / `capacitor=wilco` to match the build in review. This branch is the
post-approval cutover, ready to merge the moment Apple says yes.
